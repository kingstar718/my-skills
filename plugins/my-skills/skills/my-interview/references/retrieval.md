# 检索与内容结构

如何从 interview-wiki 库里选题、读答案。所有路径相对内容库根目录（`local_dir`），脚本用 `python scripts/xxx.py`（本机 Python 3.12）。

## 一、定位脚本 outline.py

即时生成、不落盘，是首选检索入口：

```bash
python scripts/outline.py Redis            # 文件名含 Redis 的专题标题树 + 行号
python scripts/outline.py --grep 缓存预热   # 全库定位考点：标题命中 + 正文命中(标所在小节)
python scripts/outline.py --grep -t 预热    # 只搜标题
python scripts/outline.py --tech            # 列出 44 个算法技术词 + 各自覆盖题数
python scripts/outline.py --tech 单调栈      # 该技术词下的所有题解
```

- **八股出题**：`--grep <考点>` 确认库里有没有、在哪篇哪小节；`<文件名>` 看专题结构挑主问题。
- **算法出题**：`--tech` 按细技术词找题；或直接查索引。

## 二、三大索引（content/ 根目录）

| 文件 | 内容 |
|------|------|
| `高频题目索引.md` | A. 高频算法题 Top 榜（题号/题名/专题/难度/热度/常考公司）· B. 高频八股题优先级 · C. **各公司风格与高频方向** |
| `算法题索引.md` | 全库算法题解按专题分组（gen_topics.py 生成） |
| `知识点索引.md` | 八股总览，各篇真实 H3 + 锚点（gen_index.py 生成） |

- 组卷 / 按公司出题 → 先读 `高频题目索引.md` 的 C 部分定方向，再从 A/B 取具体题。
- 索引里题名带链接即已有题解，`难度/热度/常考公司` 以题解元数据行为准。

## 三、算法题解结构（content/algorithms/problems/）

文件名 `<题号>-<英文短名>.md`（如 `01-two-sum.md`、`146-lru-cache.md`）。

```markdown
---
topics:            # 13 粗套路，决定归属哪页
  - 哈希表
  - 数组与字符串
techniques:        # 44 细技术词，决定页内分组
  - 哈希查表
---

# 1. 两数之和（Two Sum）

频次 ★★★★★ · 难度 🟢 · 高频：全厂   ← 元数据行，难度/频次/公司权威源

## 题目      ## 思路      ## 代码      ## 复杂度
## 边界条件  ## 变式      ## 易错点    ## 面试追问   ## 关联题
```

**固定九节，顺序不变**。讲解时：
- 出题只展示 `## 题目`（含约束、示例），隐藏其余。
- 讲解读 `## 思路 / 复杂度 / 边界条件 / 易错点`，代码给 `## 代码`。
- 关联题读 `## 关联题`（同套路/进阶/易混/知识点四类，含互链）。
- 元数据行是难度/频次/公司的权威源。

## 四、八股篇目结构（content/interview/<分类>/）

分类目录：`Java / 框架 / 数据库 / 中间件 / 计算机基础 / 分布式与架构 / 工程实践 / 面试`。

```markdown
# JVM

## 面试追问地图
| 主问题 | 必讲关键点 | 下一层追问 |
|--------|-----------|-----------|
| 运行时内存 | 堆、栈、方法区… | 每个区域可能出现什么 OOM |
| GC 算法 | 标记清除/复制/整理 | 分代假说、跨代引用 |
...

## 一、运行时内存与对象
### 运行时数据区有哪些？各自会出现什么 OOM？
频次 ★★★★★ · 难度 🟡
**是什么**：…
**为什么这么设计**：…
**常见追问**：…
```

**出题与追问**：
- `## 面试追问地图` 是出题源：每行「主问题」当一道题，「下一层追问」当连环深挖脚本。
- 评估用户回答对照对应小节的 `是什么 / 为什么这么设计 / 常见追问`。
- 小节标题是稳定语义 ID（问法式），可直接作锚点。

## 五、链接约定（复盘清单用）

interview-wiki 用 Quartz shortest 语义，链接**按文件名全库唯一匹配**：

- 首选纯文件名：`[JVM](JVM.md)`、`[MySQL](MySQL.md#索引)`。
- 锚点由 github-slugger 规则生成：小写 + 删标点 + 空格转 `-`（见 `scripts/slug.py`）。
- 复盘薄弱点清单给链接时用 `文件名.md#锚点`，不要用相对多段路径（`../interview/JVM.md` 会死链）。
- 这些链接是给用户回 wiki 站点查看用的，my-interview 本身不写库。
